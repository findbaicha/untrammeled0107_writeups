[Reverse] Runtime State Machine 动态生成Flag

Reverse / State Machine / Runtime / ELF 难度：困难

0x01 题目初步分析

运行 ELF 程序，输入任意内容：

echo test | ./monolith_v1

输出为一串 0/1，比特流，没有任何 flag 信息。

说明：

flag 不在静态数据中，必须触发某种条件。

0x02 静态分析

使用 IDA / Ghidra 分析程序逻辑：

state = 0x1337;

for(i=0;i<len;i++){
    state = mix(state, input[i]);

    char b = state & 1;
    write(1, &b, 1);
}

可以看出程序是一个典型的状态机（State Machine）。

0x03 核心函数 mix

uint32_t mix(uint32_t s, uint8_t x){
    s ^= x;
    s *= 0x45d9f3b;
    s ^= s >> 16;
    return s;
}

特点：

1. 非线性变换

2. 类似 hash / PRNG

3. 完全 deterministic

0x04 关键 Gate 条件

继续向下分析发现：

if((state & 0xfff0ffff) == 0x12340000){
    gen_flag(state, flag);
}

这是整个题的核心：

必须构造输入，使最终 state 满足该条件。

0x05 Flag 生成逻辑

a = state ^ 0xdeadbeef
b = mix(state, 0x42)
c = mix(b, 0x99)

最终拼接：

flag{ hex(a) + hex(b) + hex(c) }

说明：

flag 是动态生成的，不存在于二进制中。

0x06 解题思路

问题转化为：

state_n = F(input)

find input:
(state_n & mask) == target

本质是逆向状态机。

0x07 Python 求解

def mix(s, x):
    s ^= x
    s = (s * 0x45d9f3b) & 0xffffffff
    s ^= s >> 16
    return s

def dfs(depth, state, path):
    if depth == 8:
        if (state & 0xfff0ffff) == 0x12340000:
            print("FOUND:", path)
            return True
        return False

    for c in range(32, 127):
        ns = mix(state, c)
        if dfs(depth+1, ns, path + chr(c)):
            return True

dfs(0, 0x1337, "")

运行后得到满足条件的输入。

0x08 触发 Flag

将求得输入喂给程序：

echo "INPUT" | ./monolith_v1

输出：

010101010101...
flag{xxxxxxxxxxxxxxxx}

0x09 最终结论

本题不是传统 RE：

❌ 不存在硬编码 flag

❌ 无法 strings 提取

❌ 无法直接解密

而是：

✔ 构造输入驱动状态机

✔ 触发 runtime 条件

✔ 程序自行生成 flag

0x0A 总结

核心考点：

1. 状态机建模

2. 输入驱动执行

3. runtime 条件控制

4. 动态 flag 生成

一句话总结：

这题的本质不是“找 flag”，而是“让程序自己生成 flag”。
